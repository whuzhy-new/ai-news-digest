# AI 新闻日报生成器

从公开 GitHub 仓库拉取最新 AI 资讯，经过去噪 → 事件聚合 → 分类 → 格式化，输出一份结构清晰的每日日报。

---

## 文件说明

- `query_latest.py` — 数据拉取脚本，从公开 GitHub 仓库读取主 CSV 及最新快照，过滤最近 8 小时数据并自动去噪。

---

## 第一步：获取数据（已含去噪）

运行数据拉取脚本，获取最新快照且已去噪的新闻条目：

```bash
python ai-news-digest/query_latest.py --mode latest
```

> 仓库为公开仓库，无需 Token 即可访问。`--token` 参数可选，用于提高 GitHub API 限额。如需获取总表近 N 小时数据，使用 `--mode master --hours N`。

脚本已内置去噪过滤，输出即为干净数据。将输出结果解析为结构化行，字段包括：`id`、`source`、`url`、`headline`、`abstract`、`img_url`、`publish_time`。

**备选方案**：若用户直接提供 CSV 链接，使用 WebFetch 读取并解析。

---

## 第二步：事件聚合

将报道**同一新闻事件**的多条数据合并为一个事件条目。这一步至关重要——同一事件往往有多家媒体报道。

### 聚合判断信号

以下情况应当合并：

- 相同公司/人物/产品 + 相同事件类型（发布、融资、事故、政策）
- 本质相同的事实（如"DeepSeek 首次融资"与"梁文锋想通了"属同一事件）
- 同一事件的跨语言报道（如英文 Reuters + 中文量子位 报道同一话题）
- 多来源报道同一事件（如 Vercel 数据泄露被 HN + 官方 + X 同时报道）

### 聚合规则

- **标题**：选取信息量最丰富的一条作为主标题
- **摘要**：综合所有来源的摘要，提炼最完整、最具深度的一段描述
- **来源**：收集全部来源的链接与名称

### 聚合后事件结构

```
headline : 最优标题
summary  : 综合摘要（从所有来源提炼）
sources  : [(来源名, URL), ...]
```

---

## 第三步：分类

将每个事件（聚合后）归入以下 **6 个类别之一**：

### 🔴 AI 大事件
重大发布、突发新闻、里程碑事件。读者不看会后悔的内容。
> 示例：新模型发布、IPO 公告、重要政策决定、重大并购。

### 🇨🇳 国内动态
以中国为主的 AI 公司、产品或政策新闻。只要新闻主体为中国企业、中国市场或中国政府政策，均归此类。
> 强信号来源：`tmtpost_news`、`kr36_news`、`ifanr_news`、`qbitai_news`、`pingwest_news`、`wscn_news`，但任何来源的中国相关内容均可归此类。

### 🌍 海外动态
国际 AI 公司的产品、工具、功能更新与整合新闻（非中国公司）。
> 示例：Google Gemini Mac 版、Canva AI 2.0、Claude Code 更新、OpenAI Agents SDK。

### 🔬 技术前沿
研究突破、算法创新、工程实践、有技术深度的开源项目。
> 示例：新证明方法、新型架构、自进化 Agent 引擎、安全漏洞研究。

### 🎙 AI 大佬说
行业领袖与思考者的观点、洞察与经验分享。对 AI 的预测、框架或哲学思考。
> 示例：企业 AI 落地观察、职业建议、行业趋势分析、对 AI 的哲学思辨。

### ☕ 茶余饭后
轻松、有趣或生活向的 AI 话题，适合当作聊天素材。
> 示例："急诊室不能拒收病人，LLM 为什么可以拒绝回答医疗问题？"、AI 生成表情包、奇特产品功能、个人趣事。

### 分类优先级规则

1. **地域优先**：内容关于中国企业/市场/政策 → 国内动态；国际相关 → 海外动态
2. **重要性覆盖地域**：若确为重大事件（模型发布、IPO、里程碑决策），无论地域 → AI 大事件
3. **内容本质优先形式**：CEO 的意见推文 → AI 大佬说；CEO 的产品公告 → 对应地域/事件类
4. **模糊边界处理**：产品/工具发布优先归海外动态而非技术前沿；边缘趣味内容优先归茶余饭后而非直接丢弃

## 第四步：格式化输出

按以下模板输出 Markdown 格式日报：

```markdown
# AI 日报 · {YYYY-MM-DD}

## 🔴 AI 大事件

**{标题}**

{详细摘要，综合所有来源，包含背景、影响与重要性分析。}

[→ {来源1名称}]({url1}) ｜ [→ {来源2名称}]({url2})

---

{（每个事件之间用 --- 分隔，依此格式重复）}

{（该类别无内容时显示：本日暂无）}

## 🇨🇳 国内动态

**{标题}**

{详细摘要}

[→ {来源名称}]({url})

---

## 🌍 海外动态

**{标题}**

{详细摘要}

[→ {来源名称}]({url})

---

## 🔬 技术前沿

**{标题}**

{详细摘要}

[→ {来源名称}]({url})

---

## 🎙 AI 大佬说

**{标题}**

{详细摘要}

[→ {来源名称}]({url})

---

## ☕ 茶余饭后

**{标题}**

{详细摘要}

[→ {来源名称}]({url})

---

📊 共 {total} 条原始数据 → 去噪后 {after_noise} 条 → 事件聚合后 {events} 条 → 精选 {selected} 条
```

### 格式规范

| 规范项 | 要求 |
|--------|------|
| 类别排序 | 每个类别内按重要性降序排列 |
| 标题 | 加粗，保留原始语言（中文保持中文，英文保持英文） |
| 摘要 | 除「AI 大佬说」「茶余饭后」外，每条不少于 100 字；综合所有来源摘要，信息充分 |
| 多来源链接 | `[→ 来源名](url)` 之间用 ` ｜ `（全角竖线+空格）连接 |
| 单来源 | 直接写 `[→ 来源名](url)`，不加分隔符 |
| 空类别 | 显示"本日暂无" |
| 日期 | 格式为 YYYY-MM-DD，取数据中最多见的 `publish_time` 日期 |
| 每类上限 | 最多 8 条，超出时保留最重要的 |
| 事件间分隔 | 同类别内相邻事件之间用 `---` 分隔 |

---

## 第五步：统计数据

在日报底部输出以下统计：

- 原始数据总条数
- 去噪后条数
- 事件聚合后条数
- 最终精选条数
